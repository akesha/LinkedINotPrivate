The Evidence Pack

Everything Fairlinked documented is independently verifiable. Here is what they found and where you can check it yourself.

Exhibit 1

LinkedIn's JavaScript Bundle

A 2.7 MB file served to every Chrome user who visits linkedin.com. It contains a hardcoded array of 6,222 Chrome extension IDs, each paired with a specific internal file path that LinkedIn engineers mapped for detection. Also contains the scanning functions: fetchExtensions, scanDOMForPrefix, and fireExtensionDetectedEvents, which transmits results to LinkedIn's telemetry endpoint.

Any developer can verify this independently. Open linkedin.com in Chrome, open developer tools, find the JavaScript bundle, search for fetchExtensions.

Exhibit 4

Sworn Affidavit from LinkedIn's Senior Engineering Manager

Milinda Lakkam, Senior Manager of Software Engineering and Machine Learning at LinkedIn, submitted this affidavit under penalty of perjury in German court proceedings on February 6, 2026. She describes herself as responsible for LinkedIn's anti-abuse systems and confirms that LinkedIn invested in extension detection.

The contradiction in one paragraph: Lakkam states that LinkedIn's models "do not take the use of any particular browser extension(s) into account" — then states in the same paragraph that LinkedIn's systems "may have taken action against LinkedIn users that happen to have [a specific extension] installed." Both claims. Same paragraph. Signed under penalty of perjury. None of this was disclosed to users.
